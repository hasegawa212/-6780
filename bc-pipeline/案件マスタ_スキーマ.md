# 案件マスタ スキーマ（連携元定義）

BC 自動生成パイプラインが参照する入力データの定義。`/generate` には
**抽出 JSON（`extracted`）** と **案件マスタ（`deal_master`）** を渡す。

連携元（Google Sheets「案件マスタ」タブ / スプレッドシート ID
`1bDAGArxrGwKQbY8F-IZ0RcfWaoPtRB7_F9UowPyEgig`）の列名は、ここで定義する
フィールド名に揃えること。

## 1. 物件種別 `bukken`

| 値 | テンプレ | シート |
|----|----------|--------|
| `戸建`（別名: 戸建て / 一戸建 / `36-1`） | `blank_36-1.xlsx` | BC戸建 |
| `区分`（別名: 区分所有 / マンション / `37-1`） | `blank_37-1.xlsx` | BC区分 |

## 2. 抽出 JSON `extracted`（物件資料から抽出。`/extract` の出力）

| フィールド | 型 | 説明 | 例 |
|------------|----|------|----|
| `shozai` | string | 所在地（地番・住居表示） | `栃木県宇都宮市清原台二丁目` |
| `kuiki` | string | 区域区分 | `市街化区域` |
| `yoto` | string | 用途地域（正式名称に正規化） | `第1種中高層住居専用地域` |
| `nijuni_jo` | boolean | 法22条区域なら `true` | `true` |
| `kenpei` | integer | 建蔽率(%) | `60` |
| `yoseki` | integer | 容積率(%) | `200` |

> `yoto` は略称（例 `第1種中高層`）でも可。`bc_schema.normalize_yoto()` が
> 正式名称へ寄せ、`checkbox_361` / `checkbox_371` の該当欄に ■ を立てる。

## 3. 案件マスタ `deal_master`（営業・契約側の確定値）

| フィールド | 型 | 説明 | 例 |
|------------|----|------|----|
| `buyer_C` | string | 買主C（BC の売却先） | `東洋建設ホーム株式会社` |
| `bc_baibai_daikin` | integer | BC売買代金（円） | `27800000` |

## 4. 拡張について

`extracted` / `deal_master` は未知フィールドも受け付ける（`extra="allow"`）。
帳票に項目を増やす場合は `bc_schema.py` の `Cell(...)` を追加し、
白紙テンプレを `python make_blank_templates.py` で再生成する。
