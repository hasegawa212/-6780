# 運用ルール（ミスらないためのチェックリスト）

このボットを安全・確実に運用するためのルール。困ったらまずここを見る。

---

## 🟢 黄金ルール 5か条

1. **プレースホルダのまま貼らない。** `xoxb-あなたのトークン` `◯◯◯` `ghp_…` などは**必ず実際の値に置き換えてから**実行する。
2. **設定を変えたら必ず再起動。** plist を編集しただけでは反映されない。`launchctl unload → load` まで実行する。
3. **変更後は必ず `/status` で確認。** 思い込みで「できたはず」にしない。表示が変わったか目で見る。
4. **エラーが出たら文言をそのまま共有。** 自己判断で消したり省略しない。原因はだいたいエラー文に書いてある。
5. **秘密情報（トークン/パスワード）はチャットに貼ったら後で必ず再生成。** いったん漏れた前提で扱う。

---

## 🚀 コードを最新にする（デプロイ）

機能を更新したら、これを貼るだけ（1コマンド）。

```bash
cd ~/telegram-ai-bot && curl -fsSL "https://raw.githubusercontent.com/hasegawa212/-6780/refs/heads/claude/loving-pasteur-KqQsk/telegram-ai-bot/mega_bot.py" -o mega_bot.py; tg=~/Library/LaunchAgents/com.martialarts.telegram-bot.plist; launchctl unload "$tg" 2>/dev/null; pkill -9 -f mega_bot.py; sleep 3; launchctl load "$tg"; echo "✅ 再起動完了"
```

- `✅ 再起動完了` が出る → OK
- `curl` がエラー → ネット/URLの問題。少し待って再実行

## 🔑 設定（鍵）を変える

共通の書き込み関数を一度貼ってから、必要な行を実行する。**値は実物に置換**。

```bash
PLIST=~/Library/LaunchAgents/com.martialarts.telegram-bot.plist
setenv(){ /usr/libexec/PlistBuddy -c "Delete :EnvironmentVariables:$1" "$PLIST" 2>/dev/null; /usr/libexec/PlistBuddy -c "Add :EnvironmentVariables:$1 string $2" "$PLIST"; echo "set $1"; }

# 例（←の値を自分のものに）
setenv TELEGRAM_BOT_TOKEN 数字:英数字        # ← BotFatherのトークン
setenv ALLOWED_TELEGRAM_USER_IDS 123456789   # ← 自分のTelegram ID（@userinfobot で確認・複数はカンマ区切り）
setenv SLACK_BOT_TOKEN xoxb-…                # ← Slack(chat:write 等)
setenv GMAIL_ADDRESS you@gmail.com
setenv GMAIL_APP_PASSWORD "16桁アプリPW"
setenv CLAUDE_CODE_CWD /Users/yourname/work

# 最後に必ず反映
launchctl unload "$PLIST" 2>/dev/null; pkill -9 -f mega_bot.py; sleep 3; launchctl load "$PLIST"; echo "✅ 反映"
```

JSON を入れる MCP_SERVERS だけは値に "" を含むのでシングルクォートで：

```bash
/usr/libexec/PlistBuddy -c "Delete :EnvironmentVariables:MCP_SERVERS" "$PLIST" 2>/dev/null
/usr/libexec/PlistBuddy -c 'Add :EnvironmentVariables:MCP_SERVERS string [{"type":"url","name":"github","url":"https://api.githubcopilot.com/mcp/","authorization_token":"ghp_…"}]' "$PLIST"
```

## ✅ 確認のルール

設定を変えたら **Telegram で `/status`** を送り、次を目視する。

| 見る項目 | 期待する表示 |
|---|---|
| Slack | `💬 Slack送信: 利用可 / 👥 名簿 N名` |
| Claude Code | `🛠 Claude Code連携: 会話から利用可` |
| MCP | `🌐 MCP接続: 1件` |
| スケジューラ | `⏰ スケジューラ: ON（…）` |

**重要：** Slack送信・電話・/build・/code・Slack学習は **認可ユーザー限定**。
`/status` で「利用可」でも、自分の Telegram ID が `ALLOWED_TELEGRAM_USER_IDS` に入っていないと使えない。
→ `⛔ …は認可ユーザー専用 (ID: 12345)` と言われたら、その ID を `ALLOWED_TELEGRAM_USER_IDS` に登録する。

## 🔐 セキュリティ

- トークン/パスワードを**チャットに貼ったら、動作確認後に再生成**する（Telegram=BotFather `/revoke`、Slack=Regenerate、GitHub=Regenerate）。
- **銀行などのパスワードはボットに保存しない。** リンク（ブックマーク）だけ登録し、パスワードは端末の自動入力（iCloudキーチェーン等）に任せる。
- 認可ユーザーは Mac 上でコマンド実行できる＝強力。**Telegram の2段階認証をON**にする。
- `CLAUDE_CODE_CWD`（PC作業の作業フォルダ）は**必要な範囲だけ**に絞る。
- バックアップ：毎朝ブリーフィング時に全データが自動でTelegramに届く。`/export` で手動書き出しも可。

## 🆘 トラブル時の対処

| 症状 | 対処 |
|---|---|
| ボットが無反応 | 起動に数十秒。少し待つ → ダメなら再起動コマンド。トークン変更直後は特に待つ |
| `⛔ 認可ユーザー専用` | 表示された ID を `ALLOWED_TELEGRAM_USER_IDS` に登録して再起動 |
| Slack `not_in_channel` | そのチャンネルで `/invite @アプリ名` |
| Slack `missing_scope` | Bot Token Scopes に不足スコープ追加 → Reinstall |
| GitHub 権限不足 | fine-grained トークンの Repository access / Permissions を見直して再発行 |
| Claude Code 不可 | `pip3 install claude-agent-sdk` と `npm install -g @anthropic-ai/claude-code` を確認 |
| `/status` が変わらない | 再起動忘れ。`launchctl unload→load` を実行 |

**迷ったら：エラー文をそのままコピーして共有する。**

## 💬 Slack ボット（@arts）

Slack で `@arts ◯◯`（メンション）または DM すると Claude が応答する。
**Telegram ボットと「同じ脳」**：同じデータ（`~/.telegram-mega-bot/`）と同じ 52 ツール
（顧客台帳・記憶・知識・名簿・日報・予定・経費・ToDo・メール・Slack送受信・リマインダー 等）を共有する。
別プロセスだが、台帳・記憶は両方から読み書きできる（反映は各プロセスの再起動でも確実）。

- スレッド内は**履歴無制限**（スレッド全体を毎回読む・再起動でも残る）。要 `*:history` スコープ。
- 共有する台帳キーは `SLACK_BRAIN_CHAT_ID`。Telegram と同じデータにするには、自分の
  Telegram チャットID（`@userinfobot` で確認）を設定する。未設定なら Slack 独自台帳になる。
- 送信/電話/PC作業など**要認可ツール**は `SLACK_ADMIN_USER_IDS`（Slackユーザー I D・カンマ区切り）で制限。
  未設定なら社内前提で全員可。画像/動画生成は Telegram 専用（Slack では code_execution で代替）。
- `mega_bot.py` を取り込めない環境では自動で単体モード（チャット＋検索＋資料生成）に降格。

### 常駐化（再起動・ログアウトでも自動起動）

Mac のターミナルに丸ごと貼る。`ANTHROPIC_API_KEY` は Telegram の plist から流用する。

```bash
TGP=~/Library/LaunchAgents/com.martialarts.telegram-bot.plist
AKEY=$(/usr/libexec/PlistBuddy -c "Print :EnvironmentVariables:ANTHROPIC_API_KEY" "$TGP")
PY=$(command -v python3)
SP=~/Library/LaunchAgents/com.martialarts.slack-bot.plist
cat > "$SP" << PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0"><dict>
  <key>Label</key><string>com.martialarts.slack-bot</string>
  <key>ProgramArguments</key>
  <array><string>$PY</string><string>$HOME/telegram-ai-bot/slack_bot.py</string></array>
  <key>WorkingDirectory</key><string>$HOME/telegram-ai-bot</string>
  <key>EnvironmentVariables</key><dict>
    <key>ANTHROPIC_API_KEY</key><string>$AKEY</string>
    <key>SLACK_BOT_TOKEN</key><string>xoxb-…</string>
    <key>SLACK_APP_TOKEN</key><string>xapp-…</string>
    <key>CLAUDE_MODEL</key><string>claude-opus-4-8</string>
  </dict>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
  <key>StandardOutPath</key><string>$HOME/.telegram-mega-bot/slack-bot.log</string>
  <key>StandardErrorPath</key><string>$HOME/.telegram-mega-bot/slack-bot.err</string>
</dict></plist>
PLIST
pkill -9 -f slack_bot.py 2>/dev/null; sleep 2
launchctl unload "$SP" 2>/dev/null; launchctl load "$SP"
echo "✅ Slack bot 常駐化 完了"; sleep 4; tail -n 5 ~/.telegram-mega-bot/slack-bot.err
```

- `SLACK_BOT_TOKEN`(xoxb-…) / `SLACK_APP_TOKEN`(xapp-…) は**実物に置換**（黄金ルール1）。
- 反映・再起動は Telegram と同じく `launchctl unload "$SP" → load "$SP"`。
- コード更新は `curl …/slack_bot.py -o ~/telegram-ai-bot/slack_bot.py` の後に再起動。
- ログ: `~/.telegram-mega-bot/slack-bot.log` / `…/slack-bot.err`。

### つまずきポイント

| 症状 | 対処 |
|---|---|
| 反応しない（チャンネル） | そのチャンネルで `/invite @arts` |
| 反応しない（全体） | `tail ~/.telegram-mega-bot/slack-bot.err` を確認。トークン切れが多い |
| `not_authed`/`invalid_auth` | xoxb/xapp を再生成して plist 差し替え→再起動 |
| `pip` で `externally-managed` | `pip3 install --break-system-packages --user slack-bolt aiohttp anthropic` |
